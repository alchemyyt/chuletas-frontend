Mobile First

Mobile firs ya no es un sistema de diseño es casi un estandar la mayor parte de dispositivos conectados a la web son movil.
Es tan usado porque es mas facil agregar contenido que quitar y es mas facil de escalar el proyecto.

Pasos a Seguir

1)Analizando el diseño

Revisamos el diseño que creamos o que nos mandaron por ejemplo en figma.
Figma nos da una sugerencia de el tamaño de los estilos pero hay que usarlos solo de referencia

2)Arquitectura inicial

La arquitectura web se puede definir como la forma en que las páginas de un sitio web están estructuradas y enlazadas entre sí (de manera lógica y coherente). Una arquitectura web ideal ayuda a los usuarios y a los motores de búsqueda a encontrar fácilmente lo que están buscando en un sitio web.

3)Assets de nuestro proyecto

Creamos la carpeta assets y buscamos de antemano todos los iconos o logos que vamos a usar y imagenes que necesitemos.Lo podemos buscar de font awesome, google fonts o lo que nos mandaron de figma dandole click y exportandolas.
Creamos una carpeta icons y una img.
No usar espacio en los archivos

4)Fuentes de nuestro Projecto

Buscamos las fuentes que vamos a usar en google fonts o otra parte, las ponemos en el head y las importamos con css revisar en curso definitivo de CSS y HTML

5)Estilos iniciales y variables

Ponemos el margen y el padding en 0 por defecto, ponemos border-sizing border box, declaramos las variables de los colores en el root y cambiamos el tamaño de la fuente del html a 62.5% para que un rem sea 10 px y importamos la fuente

    Orden de implementar estilos

    1. Herencia SASS (si se usa): @extend, @mixin, e.g. clearfix
    2. Contenido generado(Puesto con :before :after): content
    3. Posicion y layour: positiony, z-index, top, bottom, left, right, Flexbox properties, float, clear
    4. Display y visibilidad: display, opacity, transform
    5. Recortes: overflow, clip
    6. Animacion: animation, transition
    7. Forma: margin, box-shadow, border, border-radius, box-sizing, width, height, padding
    8. Fondo: background, cursor
    9. Tipografia: font-size, line-height, font-family, font-weight, font-style, text-align, text-transform, word-spacing, color
    10. Pseudo clases y elementos: :hover,:focus,:active,:before,:after,:first-child,:last-child

9)Scroll horizontal con CSS

  • Es importantisimo poner un tamaño maximo del contenedor para que funcione el slider porque sino no se esconde con hiden si no que se adapta
  • overflow-x
    La propiedad de CSS overflow-x establece lo que se muestra cuando el contenido desborda los bordes izquierdo y derecho de un elemento a nivel de bloque. Puede que no sea nada, una barra de desplazamiento o el contenido adicional.
  • overscroll-behavior
    la propiedad de css overscroll-behabior establece lo que hace un navegador cuando alcanza el límite de un área de desplazamiento. Es una abreviatura de overscroll-behavior-x y overscroll-behavior-y.
    Se usa para evitar el rebote de las animaciones de android que pueden ser molestas y mover el slider
  • scroll-snap-type
    La propiedad CSS scroll-snap-type establece qué tan estrictamente se aplican los puntos de snap en el contenedor de desplazamiento en caso de que haya uno
